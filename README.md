# PS40 City Policy Data Collection — deploy guide

This folder is a static site + Google Apps Script backend. No build step, no framework. Follow these steps once, then share the GitHub Pages URL with students.

## 1. Create the Google Sheet (the database)

1. Go to https://sheets.new to create a new empty spreadsheet. Rename it something like `PS40 City Policy Data`.
2. In the sheet, go to **File → Import → Upload** and upload `modified_data/cities_seed.csv` (from the parent `lecture6` directory, generated by `python3 code/master.py`).
3. Choose **Replace spreadsheet** as the import location and **Detect automatically** for the separator. Click **Import data**.
4. You should now see 485 rows plus a header. Confirm the header row is exactly:
   `fips, city, state, dvs_2024, mrp_bodycam_support, assigned_at, submitted_at, bodycam_answer, bodycam_notes, nondisc_answer, nondisc_notes, zeroemiss_answer, zeroemiss_notes, student_name, student_id`

## 2. Paste in the Apps Script backend

1. In the spreadsheet, open **Extensions → Apps Script**.
2. Delete the stub `function myFunction() {}`.
3. Open `webapp/apps_script.gs` in this folder and copy its entire contents into the Apps Script editor.
4. Click the floppy-disk **Save** icon. Name the script project anything (e.g., `PS40 backend`).

## 3. Deploy the script as a web app

1. In the Apps Script editor, click **Deploy → New deployment**.
2. Click the gear icon and choose **Web app**.
3. Fill in:
   - **Description**: `PS40 backend v1`
   - **Execute as**: `Me`
   - **Who has access**: `Anyone`
4. Click **Deploy**. Google will ask you to authorize the script — click through and approve. (You may see a "Google hasn't verified this app" screen: click **Advanced → Go to [project] (unsafe)**. This is normal for your own scripts.)
5. Copy the **Web app URL** Google shows you. It will look like `https://script.google.com/macros/s/AK.../exec`.

> ⚠️ This URL is public — anyone with the link can read and write the sheet. That's fine for a class project, but don't reuse it for anything sensitive.

## 4. Wire the URL into the frontend

1. Open `webapp/app.js` in a text editor.
2. Replace `PASTE_YOUR_APPS_SCRIPT_WEBAPP_URL_HERE` with the URL from step 3.5.
3. Save.

## 5. Smoke-test the backend before deploying Pages

From a terminal:

```bash
# Should return {"ok":true,"rows":[...]} with 485 rows.
curl -L "$URL"

# Should return {"ok":true,"row":{...}} with one city, and that city's
# assigned_at should now be populated when you re-run the GET.
curl -L -X POST -d 'action=claim' "$URL"
```

If both work, you're good.

## 6. Publish to GitHub Pages

1. Create a new GitHub repository (public is simplest for Pages).
2. Copy the contents of this `webapp/` folder to the repository root (or to a `/docs` subfolder — whichever you prefer).
3. Commit and push.
4. In the repo settings, go to **Pages → Build and deployment → Source: Deploy from a branch**, pick `main` and either `/ (root)` or `/docs` to match where you put the files.
5. Wait ~1 minute, then visit the URL GitHub shows you (`https://<user>.github.io/<repo>/`). You should see the landing card with the "Get my city" button, and a "📊 Results" button top-right.

## 7. Share with students

Give students the Pages URL. They click **Get my city**, research the three policies, and submit. To watch class progress, click the **📊 Results** button in the top-right — that switches to the results view with live KPI counters and the four binscatter plots. The **🔄 Refresh** button there re-pulls the sheet on demand. You can also open the Google Sheet directly at any time.

## Resetting between terms

To reuse this setup for a new class:

1. Open the Google Sheet.
2. Select columns F through O (from `assigned_at` through `student_id`) and press Delete. The base city/votes/MRP columns (A–E) stay.

All new responses will then be collected into the cleared columns.

## Troubleshooting

- **"not enough data yet"** on a plot: fewer than 10 cities have a yes/no response for that policy. Plots render once you cross that threshold.
- **`all cities have been assigned`** error on claim: every row has a non-empty `assigned_at`. Clear `assigned_at` for any cities you want to re-open (don't clear `submitted_at` unless you want to re-open the response too).
- **Washington DC** is intentionally missing a `mrp_bodycam_support` value because DC isn't a Census "place" — plot #1 drops it automatically, but plots #2–4 still include it.
- **CORS errors in the browser console**: make sure the deployment was set to `Anyone` (not `Anyone with a Google account`) and you redeployed after any changes. Apps Script sometimes caches an old version — **Deploy → Manage deployments → edit → version: New version → Deploy**.
